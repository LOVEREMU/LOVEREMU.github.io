# 面向对象

# 理解:

  • 面向：找，拿
  • 对象：东西
  • 面向对象：找或拿东西来编程

# 设计对象并使用

  • 先设计类后创建对象

    • 类：对象共同特征的描述
    • 对象：真实存在的案例
      • 属性 —— 名词 —— 定义成员变量
      • 行为 —— 动词 —— 用方法定义

        补充
    1. 类名首字母建议大写，英文，有意义，不能使用关键字，满足标识符定义
    2. 一个代码文件可以定义多个类，但只能有一个类用 public 定义，且文件名必须是 public 修饰的类名
    3. 成员变量完整格式：修饰符 数据类型 变量名 = 初始化值（不建议初始化，因为有默认值）

# 内存机制

  1. 栈内存分配变量空间
  2. 变量储存地址，该地址由成员方法引用交给变量
  3. 变量通过地址指向方法和成员变量
  4. 变量之间的赋值传递地址，即更改指向相同地址的变量，都可以访问该地址的中的方法与成员变量

# 构造器

  • 定义：定义在类中初始化一个类的对象，并返回类的地址
  • 格式：修饰符 类名（形参列表）
    • public car (){}—— 无参数构造器 (默认存在的)：初始化对象均采用默认值，当定义了有参数构造器时，若要用无参数构造器必须重新定义一个
    • public car (){}—— 有参数构造器：初始化对象的同时，接收参数为对象赋值
  • this 关键字
    • 可以出现在构造器，方法中
    • 代表当前对象地址
    • 可用于指定访问当前对象的成员变量，成员方法，需要做到见名知义

# 封装

  • 作用：告诉我们，如何设计属性和方法
  • 原则：对象代表什么，就得封装对应的数据，并提供对应的方法
  • 好处：降低学习成本

# 如何更好的封装

  1. 用 private 修饰关键字，使得成员变量只能在当前类中访问
  2. 为每个成员变量提供配套的 public 修饰的 getter,setter 方法暴露取值和赋值

# 实体 javabean

实体类

  1. 定义：一种特殊的类
  2. 特点：
    1. 类中成员变量均为私有，并且对外提供相应的 get 和 set 方法
    2. 类中必须有一个无参数的构造器
    3. 仅仅是一个用来保存数据的 java 类，数据处理交给其他类完成，实现数据与数据业务处理相分离

# 常用 API

# 包

  1. 定义：分门别类管理不同程序
  2. 作用：有利于程序的管理和维护
  3. 注意事项：
    1. 同一个包下程序可以直接访问，其他包下则需要导包才可以访问
    2. 自己程序中调用 Java 提供的程序也需要导包，Java.lang 包下程序不需要我们导包

# java.lang.String

  1. 创建字符串对象
    1. 直接双引号得到字符串，封装字符串数据
    2. new String 创建字符串对象，并调用构造器初始化字符串
  2. 常用 String 操作方法
  3. 注意事项
    1. String 对象内容不可改变，被称为不可改变字符串变量，每次改变字符串变量实际上是产生了新的字符串对象，变量每次都是指向新的字符串对象，之前字符串的内容的确是没有改变的，因此说 String 的对象是不可变的
    2. 只要是以 "" 方式写出的字符串对象，会存储到字符串常量池中， 且相同内容的字符串只会存储一份
    3. 通过 new 方式创建的字符串对象，每 new 一次都会产生一个新的对象放在堆内存中，节约内存

# java.util.Arraylist

  1. 代表集合，集合是容器用来装数组，类似于数组，数组定长，集合可动态增长 (容器相应的方法：增删改查)
  2. 创建: Arraylist list = new Arraylist() 初始长度为 10，无参构造器
  3. 增删改查操作方法
