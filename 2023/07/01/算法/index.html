# 快速排序

时间复杂度：(nlogn)

# 思路

  1. 确定分界点
  2. 调整区间 (重点)；
    1. 设置左右指针
    2. 指针均向中间走，i 遇到比目标 x 小的就继续向下直到遇到比 x 大的就停下，j 反向走过来，同样遇到比 x 小的就停下，然后交换 i，j 所指向的数
    3. 继续向下，直到两指针相遇
  3. 递归处理左右两段

# 整数划分

  1. 应用：有单调性一定能二分，可以二分的题不一定得有单调性，常用于寻找大于或小于某个值的个数，用 sort 排完序以后可以进行查找二分，减少时间复杂度，避免超时

  2. 本质：左右区间分别属于性质的成立 / 不成立，二分寻找成立 / 不成立的边界

  3. 步骤：

    1. 划分 l+r >> 1
    2. 判断条件是否成立
    3. 调整区间

  4. 代码


    lower_bound (begin,end,num)：从数组的 begin 位置到 end-1 位置二分查找第一个大于或等于 num 的数字，找到返回该数字的地址，不存在则返回 end。通过返回的地址减去起始地址 begin, 得到找到数字在数组中的下标。

    upper_bound (begin,end,num)：从数组的 begin 位置到 end-1 位置二分查找第一个大于 num 的数字，找到返回该数字的地址，不存在则返回 end。通过返回的地址减去起始地址 begin, 得到找到数字在数组中的下标。

    用函数找到第一个大于等于 target 的下标
    int n = lower_bound(a , a + x , target) - a;//x 为数组的长度
    int n = lower_bound(a.begin() , a.end(),target) - a.begin();
    用函数找到第一个大于 target 的下标
    int n = upper_bound(a , a + x , target) - a;//x 为数组的长度
    int n = upper_bound(a.begin() , a.end(),target) - a.begin();

    在从大到小的排序数组中，重载 lower_bound () 和 upper_bound ()

    lower_bound (begin,end,num,greater<type>() ): 从数组的 begin 位置到 end-1 位置二分查找第一个小于或等于 num 的数字，找到返回该数字的地址，不存在则返回 end。通过返回的地址减去起始地址 begin, 得到找到数字在数组中的下标。

    upper_bound (begin,end,num,greater<type>() ): 从数组的 begin 位置到 end-1 位置二分查找第一个小于 num 的数字，找到返回该数字的地址，不存在则返回 end。通过返回的地址减去起始地址 begin, 得到找到数字在数组中的下标。

    用函数找到第一个小于等于 target 的下标
    int n = lower_bound(a , a + x , target,greater<int>()) - a;//x 为数组的长度
    int n = lower_bound(a.begin() , a.end(),target,greater<int>()) - a.begin();
    用函数找到第一个小于 target 的下标
    int n = upper_bound(a , a + x , target,greater<int>()) - a;//x 为数组的长度
    int n = upper_bound(a.begin() , a.end(),target,greater<int>()) - a.begin();

# pair 与离散化

pair 可以轻松实现离散化。
我们可以创建一个 pair 数组，将原始数据的值作为每个 pair 第一个变量，将原始数据的位置作为第二个变量。在排序后，将原始数据值的排名（该值排序后所在的位置）赋给该值原本所在的位置即可。

# 动态规划 (Dp)

  1. 状态表示：判断题目是几维模型，寻找模型特点进行优化 (优化是在原有的公式上做等价变形)

    2. 属性：max,min, 数量
  2. 状态计算：集合划分 (选与不选，选多少，怎么选)，分析每个状态的特点

# Tips

  1. 将一位数组下标 k 转换为 n*m 二维数组下标：
  2. 当二维数组的坐标要上下左右移动时，可以定义两个数组，代表向四个位置分别移动的状态
  3. 判断对角线上 (例如 N 皇后问题)
  4. 最大公约数的和最小公倍数的内涵
    gcd (a,b) 就是 a,b 分解质因数，所有公因子的最小幂的乘积
    lcm (a,b) 就是 a,b 分解质因数，所有公因子的最大幂的乘积
  5. 裴蜀定理
    对于任意两个整数 a 和 b，存在整数 x 和 y 使得 ax + by = gcd (a, b)，其中 gcd (a, b) 表示 a 和 b 的最大公约数。
    具体而言，如果 d 是 a 和 b 的最大公约数，那么存在整数 x 和 y 使得 ax + by = d。这个等式的一个特殊情况就是贝祖恒等式，即 d = gcd (a, b)。也可推广至多个数，a,b,c=d 之类的
  6. 同余定理
    sum [r] % k 和 sum [l-1] % k 的余数如果相等，那么 sum [r] - sum [l-1] 的差值必然是 k 的倍数 ，比如说 13 % 7 和 20 % 7 (20-13)%7 =0；
  7. 当我们需要枚举 a,b,c,d 四个数字等于 e 时，因为第四个数字可以计算出来，所以至少需要三重循环，即 O (n3)，但是这肯定是要超时的，所以就想办法优化循环的次数。一个比较好的思路是把三重循环拆成两次二重循环。在第一次二重循环中，计算一下 c2+d2, 然后记录下来，在第二次对 a 和 b 的循环中可以直接使用，而不需要再次计算。如此一来，时间复杂度就被大大的简化了。至于如何记录 c2+d2 ，则可以考虑使用哈希表，或者数组 + 二分的做法。
  8. 当需要对序列的某些特定区间进行整体修改 (例如增加或减少某个数)，可以在外部设置变量，当取出或存入区间某个元素时，直接加上外部变量即可，例如： 用优先队列维护，存入值和下标，再用一个数组cnt累计每个下标增加的和，当弹出最小的值下标为 i 时，如果此时cnt[i]不等于0，说明它实际的值需要加上cnt[i]，我们将其增加后再放回优先对列，注意需要清空cnt[i]。如果此时cnt[i]等于0，那我们就成功弹出当前最小元素，这时需要将其前一个元素和后一个元素值增加，我们需要模拟链表去记录每个元素的前后元素是谁，pre[i]表示下标为i的上一个元素是谁，ne[i]表示下标为 i 的下一个元素是谁，直到堆的元素个数只剩n-k时结束循环。不难想象，堆元素的出入次数是线性的。
  9. 整型向下取整 x/y , 整型向上取整 x/y+(x%y!=0)
  10. 当需要求某段的平均值是否大于某个数时，可以将问题转化，将该段每个数都减去这个数，那么平均值的问题就被转换为，该段的和是否大于 0，非负性判断比数值判断更容易思考一些
